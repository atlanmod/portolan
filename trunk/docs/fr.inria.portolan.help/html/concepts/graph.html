Graphs

Portolan visualizations are underlined by graph concepts. The PortolanCore metamodel embeds elements which can be easily transformed in corresponding graph concepts, and then satisfying visualization needs:

  • the view has to display a name for nodes => identify model elements (IdentifiedElement type);
  • geographical maps need to locate elements=> retrieve contents from the corresponding node (LocatedElement type)
  • graphs imply to visually rely elements together through edges => explicit binding instances (Relationship type);
  • some relations are one-way => directional binding instances (DirectedRelationship type);
  • there is interest for grouping / containment notion => items may be included in characterizations (Group and Container types);
  • each item may have related individual properties, giving ability to filter on these properties => kind of a type object mechanism (Metadata type).
