Visualization Task

The visualizations stricto sensu are external components which are realized and brought to end-users by Visualization Providers. Each visualization relies on a specific viewer and a dedicated transformation chain. This chain implies Model-Driven Interoperability and embeds the metamodels corresponding to the formats needed by the viewer. We have used existing metamodels (like GraphML, from previous projects) and we defined these which were lacking (such as the KML one).

Each chain takes the central cartography model conforming to the cartography metamodel as input, translates its contents into a model conforming to the DSL of the viewer format through a model-to-model transformation, and extracts the corresponding file in the textual format using a model-to-text extractor.

The final output is opened by the viewer. The Cartography User can then manipulate the view in order to better understand and analyze it.
