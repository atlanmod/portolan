Builder Extension

The Portolan projects can support special builder which could be added using extension mechanism.

Third parties plug-ins are free to implement new builders for their own purposes. They have to declare a builder using this Portolan builder extension (both with Eclipse builder extension) in order ot have benefit of it on every Portolan project.

Syntax

The extension point ID is: fr.inria.portolan.common.builder.

The attributes are:

  • builderId: String (REQUIRED). The ID of the corresponding registration in the Eclipse builders manager.
  • name: String (OPTIONAL). A name that describes shortly the builder.

The corresponding class must be a subclass of the org.eclipse.core.resources.IncrementalProjectBuilder class and must be declared as an Eclipse builder, using the org.eclipse.core.resources.builders extension point.

Effects

The user of Portolan application can (and should) add the Portolan nature to his data project(s). Doing that, the underlying nature register adds the builders which are declared using this extension point to the project.

The resulting effect is that the declared builder will be run on this project at each change (in an incremental manner).

Example

The Portolan application comes with a builder which purpose is registration of metamodels within Portolan projects. As a good example, this builder has the parameters below:

  • fr.inria.portolan.common.builder extension:
    • builder.builderId: fr.inria.portolan.common.build.EcoreRegistrationBuilder
    • builder.name: Ecore Registration Builder
  • org.eclipse.core.resources.builders extension (every Portolan builder must be registered as an Eclipse builder too):
    • ID: fr.inria.portolan.common.build.EcoreRegistrationBuilder
    • name: Portolan Ecore Registration Builder
    • builder.hasNature: false
    • builder.run.class: fr.inria.portolan.common.build.EcoreRegistrationBuilder
